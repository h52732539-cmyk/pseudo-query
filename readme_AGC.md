## Plan: 视频检索伪查询架构选型与优化

**推荐方案二（伪查询本身作为聚类中心）**，基于以下关键论据和针对性优化。

---

### 为什么选方案二而非方案一

#### 1. 可微性（你最关注的维度）
- **方案一的致命瓶颈**：显著性 $\alpha$ → top-k 选视频 token 作中心 — 这是**不可微操作**。原 AGC 论文依赖加权聚合中 $\alpha$ 的连续性来传播梯度，但"选哪些 token 当中心"这一步本身无梯度信号。密码本虽然可学习，但它对聚类中心的影响被 top-k 截断了。
- **方案二的突破**：聚类中心 = 伪查询，直接来自密码本生成管线，完全在计算图内。配合 soft-to-hard 退火，可实现**端到端可微训练**。

#### 2. 消除双重间接性
方案一：伪查询 → 注意力分数 → top-k 选视频 token → *这些才是*中心。伪查询对最终聚类中心只有间接影响。方案二：伪查询 → 就是中心。优化路径直接。

#### 3. 内建多样性保障
方案一的问题：视频中若两个高显著性 token 语义相似，top-m 就会选出冗余中心。密码本多样性损失管不到这里（因为中心是视频 token 不是密码本条目）。方案二中，密码本多样性损失**直接作用于聚类中心来源**。

#### 4. 伪查询的双重利用
方案一中伪查询计算完显著性后就被丢弃了——浪费。方案二中它既计算显著性，又充当聚类中心。

#### 5. 跨视频语义一致性
方案二的伪查询来自共享密码本，第 k 个聚类在不同视频中倾向捕获相似语义概念（如"动作"、"场景"），有利于检索对齐。

---

### 优化后的方案二详细设计

#### Phase A: 数据感知伪查询生成（条件化交叉注意力路由）

1. **全局语义密码本** $E \in \mathbb{R}^{K \times h}$，$K$=128~256
2. **元查询** $M \in \mathbb{R}^{m \times h}$（可学习）
3. **视频条件注入**：对初始视频 token $X$ 做注意力池化 → 视频摘要 $\bar{x}$
4. **生成**：
$$Q_\Psi = \text{CrossAttn}(M + \bar{x},\ E,\ E) \in \mathbb{R}^{m \times h}$$
   - Query = 元查询 + 视频摘要；Key/Value = 密码本；**全程可微，无 top-k**

**参数建议**：$m = 32$（默认，~97.5% 压缩率），$K = 4\text{-}8 \times m$（128~256），$|\Psi| = m$

#### Phase B: 编码与显著性计算（与原 AGC 一致）

$$[Z_X^{(L)}, Z_\Psi^{(L)}] = F_{enc}([X, Q_\Psi]; \theta)$$

$$\alpha = \frac{1}{|\Psi| \cdot H} \sum_{i \in \Psi} \sum_{\eta=1}^{H} \text{Attn}_i^{(L, \eta)} \in \mathbb{R}^n$$

#### Phase C: 伪查询作为聚类中心 + Soft-to-Hard 退火

聚类中心：$\{\mu_k\}_{k=1}^m = Z_\Psi^{(L)}$（编码后的伪查询）

**分配策略**：
- **训练初期** — 软分配（可微）：
$$w_{kj} = \frac{\exp(\cos(Z_{X,j}^{(L)},\ \mu_k) / \tau)}{\sum_{k'} \exp(\cos(Z_{X,j}^{(L)},\ \mu_{k'}) / \tau)}$$
  - 温度 $\tau$：从 1.0 线性退火至 0.1
- **训练后期 / 推理** — 硬分配（$\tau \to 0$，等效 argmax），保持语义隔离

#### Phase D: 显著性加权聚合 + 残差锚定（新增）

$$\text{AggPool}_k = \frac{\sum_{j \in G_k} \alpha_j Z_{X,j}^{(L)}}{\sum_{j \in G_k} \alpha_j}$$

$$c_k = (1 - \lambda) \cdot \text{AggPool}_k + \lambda \cdot Z_{\Psi,k}^{(L)}$$

- $\lambda$ 为可学习标量（初始 0.1）；动机：若某聚类分配到极少 token，纯聚合不稳定，混入伪查询的语义作"锚点"
- $\lambda = 0$ 时退化为标准加权聚合

#### Phase E: 训练损失

$$L = L_{retrieval} + \beta_1 L_{div} + \beta_2 L_{bal}$$

| 损失 | 定义 | 作用 |
|---|---|---|
| $L_{retrieval}$ | InfoNCE（MaxSim 分数 + batch 内负采样） | 核心检索目标 |
| $L_{div}$ | $\frac{1}{K(K-1)} \sum_{i \neq j} \max(0, \cos(E_i, E_j) - \tau_{div})$ | 防止密码本条目坍塌 |
| $L_{bal}$ | Switch Transformer 风格的 $K \cdot \sum_k f_k \cdot p_k$ | 防止聚类负载不均 |

建议 $\beta_1 = 0.1$，$\beta_2 = 0.01$，$\tau_{div} = 0.5$

### 与原 AGC 及方案一的对比

| 维度 | 原 AGC | 方案一 | **方案二（推荐）** |
|---|---|---|---|
| 伪查询来源 | 静态可学习 token | 密码本动态生成 | 密码本动态生成 |
| 聚类中心 | top-m 视频 token | top-m 视频 token | **伪查询自身** |
| 中心选择可微性 | ✗ | ✗ | **✓** |
| 聚类分配可微性 | ✗ | ✗ | **✓（soft-to-hard）** |
| 多样性控制 | 无 | 密码本级别（管不到中心） | **密码本级别 = 中心级别** |
| 伪查询利用率 | 仅显著性 | 仅显著性 | **显著性 + 聚类中心** |

### 验证策略

1. **消融实验**：移除密码本 / 残差锚定 / soft-to-hard 退火 / 多样性损失 — 逐一验证每个组件
2. **基线对比**：原 AGC、方案一、MemTok、SeqResize、H-Pool
3. **指标**：MSR-VTT R@1/R@5/nDCG@10 + 索引利用率 + 聚类分配均匀度（Gini/CV）
4. **诊断**：密码本 t-SNE 可视化、聚类 token 分配分布、$\lambda$ 训练轨迹

---

### 进一步考量

1. **迭代精化**：方案二与 Slot Attention 高度相似，可探索迭代精化（pseudo-query 经多轮 cross-attention 收敛），但增加计算开销，建议作为后续实验
2. **密码本坍塌防护**：除多样性损失外，可引入 EMA 更新或定期重初始化低使用率条目（参考 VQ-VAE 做法）
3. **推理效率**：推理时只用硬分配，开销与原 AGC 相当；密码本交叉注意力是轻量操作（$m$ 个 query × $K$ 个 key），额外开销可忽略
